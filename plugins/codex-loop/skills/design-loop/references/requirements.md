# Information sufficiency

Infer the objective, repository scope, verification, dependencies, trigger, external systems, approval boundaries, budgets, outputs, and stopping condition from the request and repository when possible.

Ask only when the answer materially changes safety, an external side effect, architecture, access, cost, or the definition of done. Bundle one to three questions. Apply visible conservative defaults for everything else.

Good defaults include manual trigger, approval on risky actions, four concurrent agents, bounded retries, two no-progress rounds, and explicit repository tests as verification.
